CNF Laravel Business Management System

Complete enterprise solution for business operations, financial management, and reporting. Built with Laravel and modern admin panel integration.

Introduction

Welcome to CNF Laravel Business Management System - a comprehensive enterprise solution designed to streamline business operations, financial management, and reporting. Built on the robust Laravel framework with modern admin panel integration.

Professional Business Solution: This system provides complete management capabilities for businesses with multiple branches, employees, terminals, and complex financial operations.

What is CNF Laravel?

CNF Laravel is a feature-rich business management system offering:

  • Multi-branch Management: Complete control over multiple business locations
  • Employee Management: Comprehensive staff management with role-based access
  • Financial Tracking: Advanced accounting, ledger management, and financial reporting
  • Job & Project Management: End-to-end job tracking and billing
  • Real-time Reporting: Comprehensive analytics and business intelligence
  • Security & Access Control: Robust role-based permission system

Installation Guide

Follow these steps to install CNF Laravel Business Management System:

1

Upload Project Files

Upload all project files to your web server, ensuring the directory structure is maintained.

# Using Git git clone your-repository-url cnf-business cd cnf-business # Or upload via FTP/SFTP # Ensure all files are uploaded to correct directory
2

Environment Configuration

Configure your environment file with database and application settings:

APP_NAME="CNF Business Management" APP_ENV=production APP_DEBUG=false APP_URL=https://yourdomain.com DB_CONNECTION=mysql DB_HOST=127.0.0.1 DB_PORT=3306 DB_DATABASE=cnf_business DB_USERNAME=your_username DB_PASSWORD=your_password # Admin panel configuration ADMIN_PREFIX=cnf
3

Install Dependencies

Install PHP dependencies and generate application key:

composer install --optimize-autoloader --no-dev php artisan key:generate
4

Database Setup

Run migrations and seed the database with initial data:

php artisan migrate --seed php artisan storage:link
5

Configure Permissions

Set proper file permissions for security:

chmod -R 755 storage chmod -R 755 bootstrap/cache
Installation Complete! Access your application at https://yourdomain.com/cnf and login with default admin credentials.

Admin Login Page

The Admin Login page allows system administrators to securely access the administrative dashboard. It offers a clean and intuitive interface designed for quick and secure authentication, enabling administrators to manage users and monitor system activities.

Login Process

To log in as an administrator, the following credentials are required:

  • Email: Enter the registered admin email address associated with your account.
  • Password: Provide your account password to authenticate and access the admin panel.

Admin Dashboard Overview

The Admin Dashboard provides an at-a-glance summary of job statistics, including Total Jobs (overall job count), Total Initialized (jobs that have been created but not yet processed), Total Processing (jobs currently in progress), and Total Completed (successfully finished jobs). It also displays daily metrics such as Today's Jobs, Today's Initialized, Today's Processing, and Today's Completed, offering administrators real-time insights into current operational performance.

The Admin Dashboard also features two insightful charts for performance tracking. The first is a Graph Chart that visualizes the number of jobs created over time, helping administrators monitor activity trends and workload patterns. The second is a Bar Chart that compares Today's Jobs with the Total Jobs completed so far, providing a clear snapshot of daily productivity against overall performance.

There‚Äôs also a set of four cards showing key metrics:

  • Total Jobs Created: Displays total jobs with comparison to last month.
  • Growth Rate: Shows performance change compared to last week.
  • Average Completion Time: Indicates average job completion with month-over-month insight.
  • Commission Paid: Displays total commissions with week-over-week comparison.

The Admin Dashboard also highlights top performers and recent activities across different categories, providing a clear overview of employee, branch, terminal, and party performance, along with the most recent jobs processed.

  • Top Employee Performance: Lists employees with the highest job counts, showing how many are processing and completed.
  • Top Branch Performance: Displays branches ranked by total jobs and their current processing and completed counts.
  • Top Terminal Performance: Highlights terminals with the most activity, including processing and completed jobs.
  • Top Party Performance: Shows parties with the highest job involvement, along with job status details.
  • Recent Jobs: Summarizes the latest jobs processed, including job name, status, and date.

Roles & Permissions System

The Roles & Permissions section allows administrators to manage user access and define system privileges. It consists of two main pages: Roles and Permissions.

Roles

The Roles page allows administrators to view and manage all existing roles in the system. Each role defines specific responsibilities and access levels. A dedicated button is available to navigate to the Create Role page, where new roles can be added.

Create Role Page

The Create Role page allows administrators to add new roles and assign specific permissions. Administrators can select permissions from various modules. A Check All Permissions option is available for quickly assigning all permissions to a role, ensuring complete access if required.

Permissions

The Permissions page allows administrators to view and manage all system permissions. Each permission defines a specific action that can be assigned to roles, controlling what users with those roles are allowed to do within the system. A button is provided to navigate to the Create Permission page for adding new permissions.

Create Permission Page

The Create Permission page enables administrators to add new permissions to the system. Administrators can define the permission name, description from here on.

User Management

The User Management module lets administrators manage all users, including admins and employees. It supports creating, editing, and deactivating accounts, assigning roles, and monitoring user activity for secure and efficient operations.

Admin Management

Manage system administrators with full control over the application. This includes viewing the Admin List and adding new admins through the Create Admin page.

Within the Admin List page, administrators can search for admins using From Date and To Date filters. The admin table includes columns such as SL, First Name, Last Name, Email, User Role, Department, etc. Additionally, pagination and search options are available for easier navigation, along with the ability to enable or disable user accounts.

Create Admin Page

The Create Admin page allows administrators to add new admin accounts, assign roles, and configure permissions for full system access.

Employee Management

Complete employee management system, including branch assignments. Administrators can view all employees in the Employee List and add new employees through the Create Employee page.

Within the Employee List page, administrators can filter employees using From Date and To Date to quickly find specific records. The employee table includes columns such as SL, First Name, Last Name, Email, User Role, Department, Can Login, Action, Created By, and Create Date. Pagination and search features are also provided for easier navigation and management.

Create Employee Page

The Create Employee page enables adding new employee accounts, assigning roles, and linking them to specific branches for effective operational management.

Route Structure

Understanding the application's route structure and URL patterns.

Route Prefix

All business management routes are prefixed with /cnf and protected by authentication:

Route::prefix('cnf')->middleware('auth')->group(function () { // All CNF routes go here // Protected by authentication middleware });

Route Naming Convention

Consistent naming pattern for all routes:

// Resource routes follow Laravel conventions Route::get('resource', [Controller::class, 'index'])->name('resource.index'); Route::get('resource/create', [Controller::class, 'create'])->name('resource.create'); Route::post('resource/store', [Controller::class, 'store'])->name('resource.store'); Route::get('resource/edit/{id}', [Controller::class, 'edit'])->name('resource.edit'); Route::put('resource/edit/{id}', [Controller::class, 'update'])->name('resource.update'); Route::get('resource/show/{id}', [Controller::class, 'show'])->name('resource.show');

Data Table Routes

AJAX routes for datatable functionality:

// Most index routes have corresponding get routes for datatables Route::post('resource/get', [Controller::class, 'get'])->name('resource.get'); // Example usage in controllers public function get(Request $request) { return datatables()->of(Resource::query())->toJson(); }

Frequently Asked Questions

Installation & Setup

Q: How do I access the admin panel after installation?
A: Access your application at https://yourdomain.com/cnf and login with the default admin credentials.
Q: Can I change the route prefix from '/cnf' to something else?
A: Yes, update the route prefix in your routes file and ensure all links are updated accordingly.

User Management

Q: How do I create different user roles?
A: Use the Roles section in the admin panel to create new roles and assign specific permissions to each role.
Q: Can employees have different access levels?
A: Yes, assign employees to different roles with specific permissions to control their access levels.

Financial Management

Q: How do I set up the chart of accounts?
A: Use the Chart of Accounts section to create account categories and sub-accounts based on your business needs.
Q: Can I track multiple bank accounts?
A: Yes, you can add and manage multiple bank accounts in the Bank Management section.
